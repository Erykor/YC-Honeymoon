## 一、这场旅行为什么这样设计

我们用三句话定下了基调:

1. **挑战节日真高峰,不是擦边**——爱丁堡 Fringe 开幕周末 (8/7-8/9 砍前 3 日只留高峰)、Sziget 5 日通票、Alba 白松露真正高峰周末 (10/31-11/1),全部撞上而非错开。Munich Oktoberfest + Salzburg + Porto 已砍。
2. **参与感 > 打卡**——6 个多日参与式 camp 是核心仪式: ★★ Connemara 海岸骑乘 6 日 + ★★ Lo Stagnone 风筝冲浪 7 日 + ★★ Malta/Gozo PADI Advanced 潜水 3 日 + ★★ Vienna Elmayer 华尔兹 3 日 + ★★ Kalymnos 攀岩 7 日 lead transition + ★★ Kornati Sea Kayak 6 日。再加 Vendemmia 葡萄踩 + 橄榄自己采 + 松露自己挖 + Via Ferrata 互相系绳。能动手的不动眼,能流汗的不坐车。
3. **节奏有张有弛**——99 天里塞了 **2 个完全休息日** (D19 Sziget 后,D36 多洛米蒂后) 和 **5 个蜜月专属升级夜**,加上各 camp 后段的自由练日,保证 14 周不崩。

## 二、14 站故事线

### 🐎 Week 1: Connemara (D1-D9) · 大西洋海岸骑乘开篇
**7/29-8/6 · 8 晚 (顺延 +3 天)**

提前 9 天到爱尔兰西海岸, 跟上 Willie Leahy's Connemara Trail 6 日点对点骑乘——金标准海岸骑乘路线。每天换 B&B 过夜, 从 Loughrea 出发, 沿 Maam Cross→Leenane 峡湾→Letterfrack 海岸→Clifden→Roundstone→Galway。**D6 Dog's Bay 1.5km 白沙滩大段 gallop** 是蜜月经典画面。

- **★★ D2-D7 6 日骑乘 (7/30-8/4)**: 25km/天, 营方提供马匹+B&B+全餐, €1,500-1,800/人
- **★ D8 Cliffs of Moher**: 莫赫断崖 8km 全程徒步
- **D9 → Edinburgh**: SNN→EDI 直飞 1h, 衔接 Fringe 开幕日

### 🎭 Week 2: 爱丁堡 (D10-D12) · Fringe 3 日精华
**8/7-8/9 · 3 晚 (砍 3 日)**

砍掉前 3 天 (城堡 + Arthur's Seat + Royal Mile), 只留 **Fringe 开幕周末**:

- **D10 周五**: ★ Fringe Opening Day + Royal Edinburgh Tattoo 8/7 周五场
- **D11 周六**: Fringe 最大密度场次 (Underbelly day pass 一天连看 3 场)
- **★★ D12 周日**: **Highland Adventure Safaris 私人 4x4 + 向导** 钻进 Lost Valley 隐谷, 走 Glencoe 主谷腹地

### 🎵 Week 3: 布达佩斯 (D13-D20) · 多瑙河上的电音岛
**8/10-8/17 · 8 晚**

Sziget 把 Óbuda 岛变成 5 天不眠的电音节。中间塞了 D16 全休 + Széchenyi 温泉。

- **D18 周六**: Sziget 主舞台最大牌之夜

### 改行程 (改数据 ≠ 改 HTML, v2 重构: single source of truth)

数据和渲染已经分离。`index.html` 只负责样式和渲染,行程内容在 [`data/`](data/) 目录;**day# / 日期 / 周几 / 数字徽章全部 runtime 计算**, 加 / 砍 / 重排站点不需要级联更新。

| 文件 | 改这里 | 关键字段 |
|---|---|---|
| [`data/stations.js`](data/stations.js)   | 14 站 + 99 天主数据 (改得最多) | `TRIP_START` (出发日) + `STATIONS[].id` (string) + `days[].id` (`d1`-`dN`) + `days[].intensity` (1-5) |
| [`data/notes.js`](data/notes.js)         | 每日 tips | 键格式 `'{stationId}.{dayId}'` 例 `'dolomites.d7'` |
| [`data/routes.js`](data/routes.js)       | 地图站间连线 | `from/to` 用 string station ID |
| [`data/budget.js`](data/budget.js)       | 预算分类 + 各站合计 | `STATION_BUDGETS` 键用 string ID |
| [`data/bookings.js`](data/bookings.js)   | 预订清单 + 截止日期 | (可选 `dayRef` 字段, 暂未启用) |
| [`data/festivals.js`](data/festivals.js) | 节日 / 重点体验 | 同上 |
| [`data/packing.js`](data/packing.js)     | 装备清单 | 平铺数组 |
| [`data/expand.js`](data/expand.js)       | **不要手动改** | runtime 把 `STATIONS` + `TRIP_START` 展开为 renderer 期望的 `day`/`date`/`week`/`station_id`/`INTENSITY`/`NOTES_BY_DAY` |

**改行程典型操作:**
- **加一站** → 在 `STATIONS` 数组里 splice 一个新 `{id: 'naples', ..., days: [...]}`; 给个 string id; routes/STATION_BUDGETS 用同样 string id 关联; **后续所有日期+编号自动重算**。
- **砍一站** → 删 STATIONS 数组里的 entry; 清 routes/STATION_BUDGETS 里相关 string id 引用。
- **改某站时长** → 编辑 `station.days[]` 加/删; 全自动。
- **跨站引用** → 在 am/pm/tips 文本里写 `{{day:stationId.dayId}}` token, expand.js 自动渲染为 `D{N}`; 想图省事直接写 `D34` 也行但站重排时会失效。
- **改完跑校验**: 在终端 `node -e "..."` 加载所有 data 文件 + 调 `expandTrip()`, 看 0 orphan token + INTENSITY 数 = 总天数。

每个文件都是 `window.XXX = [...]` 纯 JS 全局赋值,GitHub Pages 直接静态部署,本地双击 `index.html` 也能跑 (不依赖 fetch / CORS)。
